Chess Question

We are going to build a chess game. These questions are designed to specifically test your understanding of inheritance, scope, embedded loops, and if you can think in code.
You'll be asked to use psuedo code, so the language isn't important.

  1. Provided in front of you is a list of classes we have used to start organizing the information needed to get started.
    How would you complete the "piece class" and beyond?
    NOTE: If they complete the info for the piece class and stop, prod them to "keep going"

    (GRADING CRITERIA)

    1. No subclasses present and needs lots of help
    2. With a little help, they list methods and attributes for "piece class" only (no subclasses present)
    3. They list subclasses but can't fill in all the methods and attributes.
    4. They list all related attributes and methods for piece class and subclasses for each piece and both players.

    1. Can't write or explain how it works, even with help.
